Stability

When does a system behave? The unit circle criterion and tests for discrete-time stability

|p| < 1 Stability condition
BIBO Bounded-input bounded-output
○ Unit circle boundary

BIBO Stability

The definition

A discrete-time LTI system is BIBO stable (bounded-input, bounded-output) if every bounded input produces a bounded output. Formally: if |x[n]| ≤ M for all n, then |y[n]| ≤ K for all n, for some finite K.

The equivalent conditions (any one implies the others):

  1. Time domain: The impulse response is absolutely summable: Σ|h[n]| < ∞
  2. Z-domain: The ROC of H(z) includes the unit circle
  3. Pole condition (causal systems): All poles of H(z) lie strictly inside the unit circle: |pᵢ| < 1 for all i
For a causal LTI system: STABLE ⟺ all poles strictly inside the unit circle.
The unit circle is the stability boundary for discrete systems, just as the imaginary axis is for continuous systems. Think of it as a fence around a paddock. Poles are horses. If all horses (poles) are inside the fence (unit circle), the system is tame — bounded inputs give bounded outputs. If any horse escapes (pole outside the circle), the system runs wild — outputs grow without bound.

Why the Unit Circle?

The geometric argument

A pole at z = p contributes a mode pⁿ to the impulse response. The absolute value is |p|ⁿ:

  • |p| < 1: |p|ⁿ → 0 as n → ∞ (decaying — stable contribution)
  • |p| = 1: |p|ⁿ = 1 for all n (persistent — marginally stable)
  • |p| > 1: |p|ⁿ → ∞ as n → ∞ (growing — unstable contribution)

For the impulse response to be absolutely summable, every mode must decay, which requires every pole strictly inside the circle.

Marginal Stability

Poles on the unit circle

A pole at |p| = 1 produces a mode that neither decays nor grows. This is marginally stable:

  • Simple pole at z = 1: produces a constant (DC) — the system integrates
  • Simple pole at z = −1: produces alternating ±1 — oscillation at Nyquist frequency
  • Simple conjugate poles at e±jθ: produces a permanent sinusoid at frequency θ
  • Repeated pole on the unit circle: UNSTABLE — produces n·ejθn which grows linearly

Marginally stable systems are not BIBO stable (a bounded input like the unit step can produce an unbounded output through an integrator). But they're useful in practice (oscillators, integrators) when carefully controlled.

The Jury Stability Test

Testing without finding roots

For high-order polynomials, finding roots numerically can be unreliable. The Jury test determines stability algebraically from the polynomial coefficients alone.

Given the characteristic polynomial D(z) = aₙzⁿ + aₙ₋₁zⁿ⁻¹ + ... + a₁z + a₀:

Necessary conditions (quick checks)

  1. D(1) > 0: No pole at z = 1
  2. (−1)ⁿ · D(−1) > 0: No pole at z = −1
  3. |a₀| < |aₙ|: Product of pole magnitudes less than 1

If any of these fail, the system is immediately unstable — no further testing needed.

The Jury table

If the necessary conditions pass, construct the Jury array:

  1. Row 1: coefficients [aₙ, aₙ₋₁, ..., a₁, a₀]
  2. Row 2: reversed [a₀, a₁, ..., aₙ₋₁, aₙ]
  3. Row 3: Row 1 − (a₀/aₙ)·Row 2 (drop last element)
  4. Row 4: reverse of Row 3
  5. Continue until 1 element remains

The system is stable if and only if all first elements of odd-numbered rows are positive (assuming aₙ > 0).

Example: Second-Order System

D(z) = z² + a₁z + a₂

The stability conditions simplify to three inequalities (the "stability triangle"):

D(1) > 0 → 1 + a₁ + a₂ > 0
D(−1) > 0 → 1 − a₁ + a₂ > 0
|a₂| < 1

These three inequalities define a triangular region in the (a₁, a₂) plane. Any second-order system with coefficients inside this triangle is stable. The vertices of the triangle are at (−2, 1), (2, 1), and (0, −1).

The stability triangle is like a safe zone on a map. You can design any second-order filter by picking a point (a₁, a₂) — as long as it's inside the triangle, your filter is guaranteed stable. Stray outside and it blows up. The triangle's shape explains why highly resonant filters (poles close to the unit circle) are sensitive to coefficient quantisation — they sit near the triangle's edges.

Schur-Cohn Criterion

A matrix formulation

An alternative to the Jury test: form the Schur-Cohn matrix from the polynomial coefficients. All eigenvalues of this matrix have magnitude less than 1 if and only if the original polynomial is stable. This formulation is more suitable for computer implementation and connects stability testing to linear algebra.

Stability Under Quantisation

When finite precision threatens stability

In actual hardware, coefficients are represented with finite precision (e.g., 16-bit fixed point). A pole designed to be at |p| = 0.99 might be quantised to |p| = 1.01 — suddenly unstable. This is called coefficient sensitivity.

Key facts:

  • Higher-order systems are more sensitive to coefficient quantisation
  • Poles near the unit circle are most vulnerable (small coefficient changes push them across)
  • Cascade of second-order sections is preferred over direct form — keeps each section's poles well-separated from the stability boundary
  • The stability triangle for second-order sections makes quantisation effects geometrically predictable

Stability of Feedback Systems

Closed-loop transfer function

For a system with forward path G(z) and feedback path F(z):

H(z) = G(z) / [1 + G(z)·F(z)]

Stability requires that all roots of 1 + G(z)·F(z) = 0 lie inside the unit circle. This is the characteristic equation of the closed-loop system. Methods for analysing this include:

  • Root locus: Plot how poles move as loop gain varies
  • Nyquist criterion: Count encirclements of −1 by G(ejω)·F(ejω)
  • Gain/phase margins: How much gain increase or phase shift before instability

Stability as a design constraint vs. an emergent property

In classical design, we ensure stability explicitly: place poles inside the circle, check with Jury test, add gain margin. But in modern adaptive and learning-based systems, stability often emerges from training rather than being designed in. This creates a tension: emergent stability may be more flexible (the system adapts) but less provable (no formal guarantee). The Z-transform framework gives us tools to verify stability after the fact — even if the system was designed by gradient descent rather than pole placement. Perhaps the future of stability analysis is not "design for stability" but "verify stability continuously at runtime."

Runtime stability verification for learning-based systems is an active research area (Lyapunov neural networks, safe RL) without mature solutions as of 2026. The tools exist for linear verification but nonlinear guarantees remain limited.
